startSession:

Gets the session into a state that is ready for your application to make API calls.

- (BOOL)startSession:(NSError **)error

Parameters

error

Out parameter used if an error occurs while starting the session. May be NULL.

Return Value

YES if the session was started, otherwise NO.

Availability

Available in Singly iOS SDK 1.2.0 and later.

Discussion

This method will execute synchronously, waiting for each operation to complete before returning. In most cases, you will actually want to call startSessionWithCompletion: to perform these steps asyncronously.

A notification (kSinglySessionStartedNotification) is posted once the session has been started.
